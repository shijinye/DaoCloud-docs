# Release Notes for v1.2.1

  # v1.2.1

  Welcome to the v1.2.1 release of Spiderpool!

  Compared with version v1.2.0, version v1.2.1 includes the following updates.

  ***

  ## Total

  Effective pull request number: 11

  [Commits](https://github.com/spidernet-io/spiderpool/compare/v1.2.0...v1.2.1)

  ## Features

  - Add configurable HTTP request timeout for IaaS Network Provider integration. [#5661](https://github.com/spidernet-io/spiderpool/pull/5661) (@cyclinder)
  - Add configurable `vethMTU` for the coordinator `veth0` device. [#5681](https://github.com/spidernet-io/spiderpool/pull/5681) (@cyclinder)
  - Add coordinator static routes configuration support. [#5683](https://github.com/spidernet-io/spiderpool/pull/5683) (@cyclinder)
  - Add ENI/network resource device plugin support for auxiliary ENI scheduling in provider mode. [#5678](https://github.com/spidernet-io/spiderpool/pull/5678) (@cyclinder)
  - Add `vlanMode` support to VLAN CNI configuration with `auto` and `manual` modes. [#5693](https://github.com/spidernet-io/spiderpool/pull/5693) (@cyclinder)

  ## Fixes

  - Fix master NIC webhook injection when provider mode is enabled. [#5703](https://github.com/spidernet-io/spiderpool/pull/5703) (@cyclinder)
  - Fix nightly Kubernetes matrix setup retries by propagating initialization failures. [#5669](https://github.com/spidernet-io/spiderpool/pull/5669) (@Copilot)
  - Fix Kruise setup retries by cleaning up the existing untar directory before `helm pull --untar`. [#5672](https://github.com/spidernet-io/spiderpool/pull/5672) (@Copilot)

  ## Tests

  - Add E2E coverage for sub-ENI capacity exhaustion and combined sub-ENI/master NIC resource injection. [#5703](https://github.com/spidernet-io/spiderpool/pull/5703) (@cyclinder)
